# UI Design Choice -- Automated Fraud Detection System

## 1. Introduction

The user interface (UI) of a software product is responsible for
enabling interaction between the user and the system.\
In the **Automated Fraud Detection System**, the UI is designed to help
analysts, administrators, and users interact with fraud monitoring tools
easily and efficiently.

This project primarily uses a **Menu-Based Interface combined with a
Direct Manipulation Interface**.

------------------------------------------------------------------------

## 2. Menu-Based Interface

A **Menu-Based Interface** allows users to interact with the system by
selecting options from menus instead of typing commands.

### How it is used in this project

In the **Analyst Dashboard**, a sidebar navigation menu provides options
such as:

-   Dashboard Overview
-   Transaction Monitoring
-   Fraud Alerts
-   Reports and Analytics
-   User Management

The analyst can click any option to navigate to that section.

### Why we use it

The Menu-Based Interface is used because:

-   It is **easy to understand and use**.
-   Users do not need to remember commands.
-   It provides **clear navigation between system modules**.
-   It reduces the chances of user errors.

This makes the system suitable for analysts who need quick access to
different monitoring tools.

------------------------------------------------------------------------

## 3. Direct Manipulation Interface

A **Direct Manipulation Interface** allows users to interact directly
with graphical elements such as buttons, charts, tables, and forms.

### How it is used in this project

In the Fraud Detection System:

-   Analysts interact with **buttons** to view transaction details.
-   **Charts and graphs** display fraud statistics and patterns.
-   **Tables** show transaction records and suspicious activities.
-   **Filters and search tools** help analysts quickly find specific
    transactions.

### Why we use it

Direct manipulation improves usability because:

-   It makes the system **interactive and intuitive**.
-   Users can understand information visually through **charts and
    graphs**.
-   Analysts can quickly **identify fraud patterns**.
-   It improves **efficiency in monitoring suspicious activities**.

------------------------------------------------------------------------

## 4. Justification of the Chosen UI

The combination of **Menu-Based Interface** and **Direct Manipulation
Interface** is suitable for the Automated Fraud Detection System
because:

1.  It provides **simple navigation** through the system modules.
2.  Analysts can easily monitor transactions using **visual
    dashboards**.
3.  Interactive elements like charts and tables make **fraud analysis
    faster**.
4.  The interface is **user-friendly and requires minimal technical
    knowledge**.

------------------------------------------------------------------------

## 5. Conclusion

The Automated Fraud Detection System uses a **Menu-Based Interface with
Direct Manipulation features** to provide an efficient, interactive, and
easy-to-use platform for monitoring and analyzing fraudulent
transactions.
